# Jira Ticket Template Skill

Standardized templates for high-quality Jira issue creation. This skill ensures consistency across teams by providing structured formats for various issue types, reducing ambiguity and improving developer productivity.

## Overview

The **Jira Ticket Template Skill** assists users in generating well-structured, professional Jira tickets. It follows industry best practices (e.g., INVEST criteria for stories, clear reproduction steps for bugs) to ensure that every ticket created is actionable and easy to understand.

## Features

- **Structured Issue Types**: Templates for Bugs, Stories, Tasks, and Epics.
- **Markdown Templates**: Pre-formatted structures for standard Jira fields such as Summary, Environment, Steps to Reproduce, Acceptance Criteria, etc.
- **Best Practices**: Adheres to industry conventions like the "As a... I want... So that..." format for user stories.

## Usage

To utilize this skill, invoke the appropriate template for the desired issue type.

### Example: Creating a User Story

**Prompt:** "Create a user story for adding a 'Forgot Password' link to the sign-in page."

**Generated Output:**
```markdown
**Summary**: [Concise title of the story]

**Description**:
As a [user role],
I want [feature or functionality],
So that [benefit or value].

**Acceptance Criteria**:
- [Criterion 1]
- [Criterion 2]
- [Criterion 3]
```

### Example: Creating a Bug Report

**Prompt:** "Create a bug template for a login failure on production."

**Generated Output:**
```markdown
### Summary
[Brief description of the bug]

### Environment
- **Device/OS:** [e.g., macOS 14.2]
- **Browser:** [e.g., Chrome 120]
- **Environment:** [e.g., Staging/Production]

### Steps to Reproduce
1. [Step 1]
2. [Step 2]
3. [Step 3]

### Expected Result
[What should have happened]

### Actual Result
[What actually happened]
```

## Installation

1. Ensure the `skills/jira-ticket-template/` directory exists in your project.
2. Verify that `agent.py` is configured to load skills from the `skills/` directory.
3. Restart the agent to apply changes.

## Prerequisites

- Jira server configured and accessible (if using direct integration).
- User permissions to create issues in the target project.
- A clear understanding of the feature or bug to be documented.
